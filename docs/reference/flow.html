    zero.flow¶

    Smart Python loops.

    Flow¶

    class zero.flow.Flow(loader)[source]¶

    Smart wrapper for iterables.

    Flow simplifies managing loops, especially for typical deep learning scenarios (it is usually used to wrap train_dataloader or any other data source).

    Flow:

    • simplifies management of the “epoch” and “iteration” variables

    • allows to customize the size of epoch

    • allows to change the underlying data loader on the fly

    • enables useful patterns

    • (not implemented: issue) allows to dump and restore loop’s state: epoch, iteration, etc.

    Parameters

    loader – any kind of iterable (DataLoader, list, iterator, generator, …)

    Raises

    AssertionError – if loader is not an iterator and is empty

    Examples

    flow = Flow([0, 1, 2, 3])
flow = Flow(range(10))
import itertools
flow = Flow(itertools.repeat(0))

    flow = Flow(torch.utils.data.DataLoader(...))

    Tutorial

    Let’s revise the conventional approach without Flow:

    loader = DataLoader(...)
iteration = 0
for epoch in range(max_epoch):
    if need_custom_epoch_size():
        assert False, 'It is possible, but not convenient'

    for x in loader:
        iteration += 1
        print('Epoch:', epoch, 'Iteration:', iteration)
        ...

    if need_new_loader():
        assert False, 'It is possible, but not convenient'

    There are several ways how you can use Flow to enhance this loop. Let’s start with creating a flow:

    flow = Flow(DataLoader(...))

    The dataloader is accessible via Flow.loader. Firstly, let’s reproduce the loop above:

    flow = Flow(DataLoader(...))
while flow.increment_epoch(max_epoch):
    for x in flow.data():
        print('Epoch:', flow.epoch, 'Iteration:', flow.iteration)

    We see that there are Flow.epoch and Flow.iteration and the latter one is created and incremented automatically.

    As for the loop’s type, while is chosen instead of for, because it

    • needs no changes when starting from non-zero epoch (for example, when resuming after loading a checkpoint)

    • enables flexible termination patterns by adding additional conditions

    For example, with zero.progress.ProgressTracker early stopping can look like this:

    while not progress.fail and flow.increment_epoch():
    ...

    In order to customize the epoch size, pass the size to Flow.data:

    while flow.increment_epoch(max_epoch):
    for x in flow.data(custom_epoch_size):
        ...

    Changing the underlying loader on the fly is possible at any moment via Flow.set_loader. For example:

    while flow.increment_epoch(max_epoch):
    for x in flow.data(custom_epoch_size):
        ...
        if need_new_loader():
            flow.set_loader(new_loader)

    Additionally, two new forms of infinite loop become possible:

    for x in flow.data(math.inf):
    ...
    if flow.iteration % frequency:
        ...

while True:
    x = flow.next()
    ...
    if flow.iteration % frequency:
        ...

    Note

    For better technical understanding, keep in mind that Flow simply incapsulates an “infinite iterator” that is constantly moving forward. The behavior is absolutely the same for both finite and infinite iterables and can be expressed with the following loop:

    while True:
    for item in loader:  # loader which is passed in the constructor
        ...

    Documentation of Flow.next and Flow.data provide helpful examples.

    ManualFlow¶

    class zero.flow.ManualFlow(*args, **kwargs)[source]¶

    Like Flow, but with additional fine-graded control.

    ManualFlow can be useful when one logical step does not correspond to one iteration (for example, you collect data from several iterations to build one training batch). The class inherits from Flow and adds some features (see documentation for details).
